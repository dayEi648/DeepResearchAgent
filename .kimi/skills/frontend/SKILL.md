# Frontend-Design



## Design Thinking

- Before coding, please ensure that you have understood the context and clarified the development intention, and then determine the aesthetic design direction:

  - **Purpose**: What problem does this interface solve? Who uses it?
  - **Style**: Using the DESIGN document in the "references" folder as the primary style, we will determine the aesthetic direction in conjunction with the project's functionality. Additionally, there are many other styles : minimalist style, organic/natural style, luxurious/refined style, playful/toy style, magazine/editorial style, decorative art/geometric style, pastel/pink style, industrial/practical style, etc. You can choose some of them to appropriately integrate into our main style (or not).
  - **Constraints**: Technical requirements (framework, performance, accessibility).
  - **Differentiation**: What makes this UNFORGETTABLE? What's the one thing someone will remember?

**CRITICAL**: The key is intentionality, not intensity.

Then implement working code (HTML/CSS/JS, TS, Vue, etc.) that is:

- Production-grade and functional
- Visually striking and memorable
- Cohesive with a clear aesthetic point-of-view
- Meticulously refined in every detail


## Frontend Aesthetics Guidelines

Focus on:
- **Primary Reference**: Primarily refer to the documents listed in "references" (located at `.kimi/skills/frontend/references/`).
- **Typography**: Choose fonts that are beautiful, unique, 和 interesting. Avoid generic fonts like Arial and Inter; opt instead for distinctive choices that elevate the frontend's aesthetics; unexpected, characterful font choices. Pair a distinctive display font with a refined body font.
- **Color & Theme**: Commit to a cohesive aesthetic. Use CSS variables for consistency. Dominant colors with sharp accents outperform timid, evenly-distributed palettes.
- **Motion**: Use animations for effects and micro-interactions. Prioritize CSS-only solutions for HTML. You can utilize various compatible Motion libraries if you find the effects satisfactory. Pay attention to high-impact moments: a carefully choreographed page loading brings more joy through staggered display (animation delay) than scattered micro-interactions. Use scroll triggering and hover states to create surprises.
- **Spatial Composition**: Unexpected layouts. Asymmetry. Overlap. Diagonal flow. Grid-breaking elements. Generous negative space OR controlled density.
- **Backgrounds & Visual Details**: Create atmosphere and depth rather than defaulting to solid colors. Add contextual effects and textures that match the overall aesthetic. Apply creative forms like gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, decorative borders, custom cursors, 和 grain overlays.

NEVER use generic AI-generated aesthetics like overused font families (Inter, Roboto, Arial, system fonts), cliched color schemes, predictable layouts and component patterns, 和 cookie-cutter design that lacks context-specific character.

Interpret creatively . No design should be the same .


**IMPORTANT**: Match implementation complexity to the aesthetic vision.  Elegance comes from executing the vision well.
